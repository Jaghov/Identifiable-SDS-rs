# M5 milestone — Optional NeuralMSM warm-start

**PRD:** [§9 Milestones](PRD-burn-port.md#9-milestones) (row **M5**), [§5.3 Optional NeuralMSM](PRD-burn-port.md#53-optional-neuralmsm).

**Status:** merged — **`snlds-msm`** crate + **`snlds-train --msm-init`** ship a simplified Burn-native warm-start path; documented simplifications below.

---

## Goal

Port **optional** **NeuralMSM**-style pretraining and **parameter transfer** into **VariationalSNLDS** so training matches the spirit of Python [`NeuralMSM.py`](../identifiable-SDS/models/NeuralMSM.py) + [`train_snlds.py`](../identifiable-SDS/train_snlds.py) init paths: fit on **PCA-reduced** observations, then copy **transitions**, **`q_logits`** / **`pi_logits`**, **`init_mean`**, **`init_cov_factor`** (and emission cov factor) into the main model before **[M4](M4.md)** training. Gate behind **CLI / config** flags so default runs do **not** require MSM.

---

## Implementation summary (merged)

| Piece | Location / notes |
|-------|------------------|
| Crate | **`crates/snlds-msm`** (lib only) |
| **PCA** | [`pca::pca_fit_transform`](../crates/snlds-msm/src/pca.rs) — **`linfa`** + **`linfa-reduction`** 0.8 (`Pca::params(dim_latent).fit(&dataset)` → `Transformer::transform`) on flattened obs `[N*T, dim_obs]`, reshaped to `[N, T, dim_latent]` |
| **NeuralMSM analogue** | [`msm::NeuralMsm`](../crates/snlds-msm/src/msm.rs) — Burn `Module` with K transition MLPs (softplus), `q_logits`, `pi_logits`, diagonal `init_cov_factor`, diagonal `emission_cov_factor`. **`marginal_log_likelihood`** uses [`snlds_core::hmm::log_forward`](../crates/snlds-core/src/hmm.rs); **`fit`** is plain Adam minibatch (no EM, no closed-form Q update) |
| **Transfer** | [`transfer::transfer_into_snlds`](../crates/snlds-msm/src/transfer.rs) — copies `transition_nets`, `q_logits`, `pi_logits`, `init_mean`, `init_cov_factor`, `emission_cov_factor` into a fresh **[`VariationalSnlds`](../crates/snlds-model/src/model.rs)** instance |
| **CLI orchestration** | [`snlds_train::warm_start`](../crates/snlds-train/src/warm_start.rs) — **`--msm-init`** triggers `--msm-restarts` random restarts, keeps the best by mean log-likelihood, then transfers; **`build_model_config`** + **`train_with_model`** support the externally initialised model |

### Intentional simplifications vs Python

| Concern | Python `NeuralMSM` | Rust `snlds-msm` |
|---------|---------------------|------------------|
| Initial / emission covariance | full `Cholesky` factor `[K, D, D]` | **diagonal** `[K, D]` factor (matches `VariationalSnlds` shape — direct copy) |
| Transition activation | `cos` (configurable) | **`softplus`** (matches the SNLDS transition MLPs we warm-start) |
| Lag | configurable (default 1; `CausalMLP` for >1) | **lag = 1** only |
| Optimisation | Adam minibatch + closed-form `Q` from paired marginals + early-stop / scheduler | **Adam minibatch** maximising marginal log-likelihood (gradient updates to `q_logits` / `pi_logits` directly); no early-stop / LR schedule |
| Gradient clipping | optional `clip_grad_norm` | not yet (relies on Adam's natural stability for warm-start) |

These keep the **Burn-native** code small while still seeding the SNLDS optimiser closer to a sensible HMM mode than random init.

## CLI

```sh
snlds-train --data-dir <dir> --output-dir <dir> --msm-init \
            [--msm-restarts <n>]    # default 3
            [--msm-epochs <n>]      # default 30
            [--msm-batch-size <n>]  # default 32
            [--msm-lr <f>]          # default 7e-3
            [--msm-hidden-dim <n>]  # default 16
            …                       # plus the M4 flags
```

`--msm-init` is **off by default** — the M4 default training path is unchanged.
