## Goal

Rust crate **`snlds-data`** that reproduces the vector observation pipeline from [identifiable-SDS/generate_data_and_train_snlds.py](../identifiable-SDS/generate_data_and_train_snlds.py): Markov discrete states, latent AR dynamics (**`cosine`** and **`poly`** `data_type`), batched leaky-ReLU emission (**`func_leaky_relu_batch`**), train/test splits. Persist outputs as **`.safetensors`** (+ JSON metadata) — not `.npy` by default ([PRD §8.1](PRD-burn-port.md)).

---

## SafeTensors key schema (v2 — replaces v1)

v1 stored **`states_*` as `F32`** (mirroring an accidental Python `float64` layout). **v2** uses **`I32`** for discrete state indices so dtype matches semantics. **`schema_version`** in **`metadata.json`** is **2** for v2 files.

Single directory per run, e.g. `out/run_name/`:

| Key | Dtype | Shape | Description |
|-----|-------|-------|-------------|
| `latents_train` | **F32** | `[N,T,dim_latent]` | |
| `latents_test` | **F32** | `[N_test,T,dim_latent]` | `N_test = max(1, num_samples // 10)` |
| `obs_train` | **F32** | `[N,T,dim_obs]` | |
| `obs_test` | **F32** | `[N_test,T,dim_obs]` | |
| `states_train` | **I32** | `[N,T]` | discrete indices `0 .. num_states-1` |
| `states_test` | **I32** | `[N_test,T]` | same |

Sidecar **`metadata.json`**: `{ "seed", "num_states", "dim_obs", "dim_latent", "seq_length", "num_samples", "sparsity_prob", "data_type", "degree", "schema_version": 2 }`.

## Implementation / SafeTensors IO

**`encode_safetensors`** in **`snlds-data`** ([`io.rs`](../crates/snlds-data/src/io.rs)) builds six per-tensor staging **`Vec<u8>`** buffers from **`numpy`-style ndarray** views and wraps them in **`TensorView` borrows** that live only for **one** **`safetensors::serialize`** call. Payload bytes are copied into the output file buffer; staging memory is freed when **`encode_safetensors`** returns. There is **no `Box::leak`** and no process-global accumulation, so calling **`save_train_test`** repeatedly (e.g. multi-seed batch generation) stays bounded.

**`metadata.json`** round-trip: **`Manifest`** implements **`Serialize`** and **`Deserialize`**; use **`load_manifest`** to read the same schema back for training scripts.

## Testing requirements

Inherit **M0** gates: `cargo fmt --check`, `cargo clippy --workspace --all-targets -D warnings`, `cargo test --workspace`.

| Area | Requirement |
|------|--------------|
| **`get_trans_mat`** | Exact **0.9 / 0.1** cyclic matrix vs [transitions.py](../identifiable-SDS/utils/transitions.py) |
| **`num_params`** | `comb(dim_latent+degree, degree)` matches `C(n+k,k)` small cases |
| **Determinism** | Same **seed** + config → identical byte tensors (ChaCha8) |
| **`generate_train_test`** | Shapes **`[N,T,·]`** / **`[N,T]`**; **`N_test = max(1, N//10)`**; latents/obs finite; **`states_* ∈ [0, num_states)`** as **`i32`** — integration covers **cosine** and **poly** (**`generate_train_test_*_shapes_ranges_and_finite`**) |
| **SafeTensors** | Write → read round-trip **all six** keys; **`latents_*`/`obs_*` `F32`**, **`states_*` `I32`** |
| **`metadata.json`** | **`schema_version`: 2** for v2 layout; typed load via **`Manifest` + `Deserialize`** (**`load_manifest`**) |
| **Poly expand** | dim=2, degree=2: length **6** features, order matches sklearn `[1,x0,x1,x0²,x0x1,x1²]` |

**Out of scope:** image pipeline, softplus dynamics, Python cross-check in CI (optional local script).

## Updates

Training-side polish for **`snlds-data`** (see **document history**):

| Item | Description |
|------|-------------|
| **`Manifest` round-trip** | **`Manifest`** derives **`Deserialize`**; **[`load_manifest`](../crates/snlds-data/src/io.rs)** reads **`metadata.json`** with the same schema as **`save_train_test`**. |
| **Polynomial rollout** | **`poly_mean_for_state`**: scalar path for one **`z_t`** and discrete state; **`poly_means_rows`** delegates row-wise; **[`roll_sequences`](../crates/snlds-data/src/generate.rs)** no longer allocates a fake **`(1, dl)`** batch every step. |
| **Emission hidden width** | **`EMISSION_HIDDEN_DIM`** (**`8`**, Python-parity) replaces magic **`8`** for leaky / cosine parameter shapes in **`transitions`** + **`generate`**. |
| **Poly integration gates** | Same shape/finite/state-range assertions as cosine (**`integration.rs`**). |
