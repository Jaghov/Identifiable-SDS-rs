# M4 milestone — Training CLI, optimizers, checkpoints

**PRD:** [§9 Milestones](PRD-burn-port.md#9-milestones) (row **M4**), [§5.2 Model / training](PRD-burn-port.md#52-model--training), [§7 Technical design](PRD-burn-port.md#7-technical-design-high-level).

**Status:** planned — checkboxes open until implementation merges.

---

## Goal

Deliver a **user-facing training path** on top of **[M3](M3.md)** model core: load **[M1](M1.md)** (or compatible) tensors, run **minibatch** training with **Adam** (Burn `optim`), **learning-rate schedule** (StepLR-style parity where documented), **gradient clipping**, **multi-restart** or configurable trial loops as needed, and **checkpoint** save/load suitable for resume and for documenting runs. **Encoder default** may remain **`factored`** until **recurrent** lands, or follow **[README](../README.md)** once updated.

Expose **logged metrics** consistent with PRD [§5.2](PRD-burn-port.md#52-model--training): ELBO, MSE / reconstruction, MSM term, temperature when annealed — feed **Rerun** via **[M-Viz+](M-Viz+.md)** when enabled.

---

## Completion checklist

- [ ] **Binary or `clap` CLI** (workspace crate e.g. `snlds-train` — name TBD): `train` entrypoint with seeds, paths, hyperparameters surfaced.
- [ ] **Data path:** read **SafeTensors** + **metadata** ([M1](M1.md) schema); batch shuffle; device/backend selection aligned with **M0** CI (**CPU**) and optional GPU features.
- [ ] **Optimizer:** Adam + hyperparams; **StepLR** (or documented analogue) configurable.
- [ ] **Gradient clipping** threshold configurable; matches documented Python behavior within stated tolerance.
- [ ] **Checkpoints:** save/load **model state** (Burn `Recorder` / native **or** **safetensors** + manifest — resolve **§12 open question** and document choice here in **Updates**).
- [ ] **Early-stop** or convergence hooks where PRD / Python scripts define thresholds (document simplifications).
- [ ] **README** training example: one **copy-paste** default aligned with **[M1](M1.md)** tiny or standard synthetic config.
- [ ] **M-Viz+** integration optional flag (`--viz`, env) — need not block M4 merge if stubbed, but **document** state in **Updates**.

## Testing requirements

Inherit **M0** gates: `cargo fmt --check`, `cargo clippy --workspace --all-targets -- -D warnings`, `cargo test --workspace`.

| Area | Requirement |
|------|-------------|
| **Smoke train** | **≥1** training step (or micro-epoch) on **CPU**, **finite** loss, no panic |
| **Checkpoint** | Save then load and **resume** one step; **key tensors** unchanged within float tolerance |
| **CLI** | `--help` documents main flags; integration test may spawn short run on temp dir |

**Out of scope (unless promoted):** **NeuralMSM** warm-start — **[M5](M5.md)**; **image** training — **[M6](M6.md)**.
