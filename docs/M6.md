# M6 milestone (stretch) — Image / video encoder–decoder path

**PRD:** [§9 Milestones](PRD-burn-port.md#9-milestones) (row **M6**), [§3 Non-goals](PRD-burn-port.md#3-non-goals-initial-release).

**Status:** **active** — promoted from deferred (2026-04-30) as the data-flow template for the planned `FlowSNLDS` encoder. The CNN itself is not the headline; the load-bearing artefact is the `EncoderKind` enum dispatch, so a third (`Flow`) variant lands with no churn to `model.rs`.

---

## Goal

Provide a **Burn**-native **CNN** (or comparable) **encoder–decoder** path for **image** / **video** observation pipelines that mirror deferred Python functionality: e.g. `encoder_type='video'`, **BouncingBallDataLoader**, and OpenCV-style rendering — **without** blocking **[M1](M1.md)–[M5](M5.md)** vector pipelines. This milestone **re-opens** PRD **non-goals** for CNN/image only after core parity is stable.

---

## Completion checklist

- [x] **Scope decision** recorded in plan `tidy-doodling-mccarthy.md`: faithful port of Python `CNNFastEncoder` / `CNNFastDecoder`; `res ∈ {16, 32, 64}` powers of two ≥ 16; latent-ball synthetic frames via reused `draw_sequence`.

## Out of scope

- **Pixel-perfect** match to Python OpenCV rendering — `draw_sequence` already matches the M-Viz GT visualiser; that's enough for M6.
- **Tuning** of the CNN architecture beyond Python parity — M6 is a data-flow template for the planned `FlowSNLDS` encoder, not a competitive image model.
- **Heavy CI gates** (large image batches, GPU runs) — revisit only if regressions surface.

---

## Downstream

Experimental **image** SNLDS demos; may reuse **[M-Viz](M-Viz.md)** **Image** logging patterns (**`snlds/render/frame`** from **M-Viz** is **latent-ball** rendering, not CNN training).

## Updates

| Item | Description |
|------|-------------|
| `EncoderKind` enum | Added to `SnldsConfig` with `Mlp` (default) and `Cnn { res }` variants; `forward()` dispatch is variant-agnostic so a future `Flow` variant is purely additive. |
| `cnn::CnnEncoder` / `cnn::CnnDecoder` | Faithful port of `CNNFastEncoder` / `CNNFastDecoder` (Python `identifiable-SDS/models/modules.py`). `n_layers = log2(res / 8)`; encoder bottoms out at `hidden × 4 × 4`, decoder lifts off `hidden × 8 × 8`. Trailing `(0,1,0,1)` pad after each `ConvTranspose2d` mirrors `F.pad`. |
| `ObservationKind::Image { res }` | Added to `GenConfig` (`snlds-data`); routes the synthetic 2-D ball trajectory through `draw_sequence` → flat NHWC RGB. `draw_sequence` itself moved from `snlds-viz` to `snlds-data` (re-exported from viz). |
| `train_config.json` v2 | Schema bumped to 2 with required `kind` field. v1 snapshots are intentionally not loadable — no production checkpoints exist yet. |
| `--encoder` / `--res` flags | Added to `snlds-train`; `--encoder cnn` requires `--res`, `--res` rejected with `--encoder mlp`. |
