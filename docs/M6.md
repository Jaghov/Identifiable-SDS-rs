# M6 milestone (stretch) — Image / video encoder–decoder path

**PRD:** [§9 Milestones](PRD-burn-port.md#9-milestones) (row **M6** stretch), [§3 Non-goals](PRD-burn-port.md#3-non-goals-initial-release).

**Status:** stretch / not scheduled — checklist **planned** until scope is promoted.

---

## Goal

Provide a **Burn**-native **CNN** (or comparable) **encoder–decoder** path for **image** / **video** observation pipelines that mirror deferred Python functionality: e.g. `encoder_type='video'`, **BouncingBallDataLoader**, and OpenCV-style rendering — **without** blocking **[M1](M1.md)–[M5](M5.md)** vector pipelines. This milestone **re-opens** PRD **non-goals** for CNN/image only after core parity is stable.

---

## Completion checklist

- [ ] **Scope decision** recorded in PRD: which **dataset(s)** and **resolution** first.
- [ ] **Encoder / decoder** `Module`s on top of **Burn** backends (CPU baseline; GPU as available).
- [ ] **Data loader** for synthetic **video** or **image** sequences; optional **Rust** **`image`** / **`png`** deps ([PRD §8.3](PRD-burn-port.md#83-optional--milestone-specific)).
- [ ] **Training integration** with **[M4](M4.md)** or successor crate; feature flags to keep default **vector** build lean.
- [ ] **Tests:** small synthetic clip or frame batch; **forward finite**; optional qualitative **Rerun** `Image` logging (**[M-Viz+](M-Viz+.md)**).
- [ ] **Docs:** README + PRD **§4 / §3** updates when promoted from non-goal.
