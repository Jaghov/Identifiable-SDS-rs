# M2 milestone — HMM kernels (forward–backward, local evidence)

**PRD:** [§9 Milestones](PRD-burn-port.md#9-milestones) (row **M2**).

**Status:** planned — checkboxes open until implementation merges.

---

## Goal

Add **discrete-state switching** primitives in **[`snlds-core`](../crates/snlds-core/)** (Burn on the **M0** CPU backend), aligned with PyTorch [`VariationalSNLDS`](../identifiable-SDS/models/VariationalSNLDS.py): **log-domain** forward messages (`log_alpha`), per-timestep normalization (`log_Z`), and backward messages (`log_beta`), given **log local evidence** \(\log p(z_t \mid s_t{=}k, \ldots)\) stacked as a tensor over batch, time, and discrete state \(K\). Use the same **temperature** scaling and **softmax-of-\(Q\)** pattern as `_alpha` / `_beta` (including `logsumexp` stabilization).

**Local evidence:** Either treat **log local evidence** tensors as **provided** (tests start from fixed **hand-checked** or **exported** values), or implement the analogue of **`_compute_local_evidence`** (Gaussian chain on latents \(z\)) and compare outputs to reference tensors — the completion checklist below lists both trajectories so parity work can land in phases (fixtures-first vs end-to-end).

See also PRD [§4 Scope](PRD-burn-port.md#4-scope) (VariationalSNLDS **Inference** row), [§6 Non-functional requirements](PRD-burn-port.md#6-non-functional-requirements) (numerical stability / HMM tests), and [§10 Success criteria](PRD-burn-port.md#10-success-criteria) item **2** (kernel match to reference).

---

## Tensor layout (informative, Python reference)

Shapes follow [`VariationalSNLDS._compute_local_evidence` / `_alpha` / `_beta`](../identifiable-SDS/models/VariationalSNLDS.py) (log domain throughout):

| Symbol / name | Shape | Role |
|---------------|-------|------|
| `local_evidence` (log) | `[N, T, K]` | Log-prob of latent step \(t\) under state \(k\) (including \(t{=}0\) init term). |
| `log_alpha` | `[N, T, K]` | Forward messages (normalized per row by `log_Z`). |
| `log_Z` | `[N, T]` | Per-timestep log normalizers. |
| `log_beta` | `[N, T, K]` | Backward messages. |

**Transition matrix:** Python stores **logits** and applies `(Q / temperature).softmax(-1).log()` with broadcasting over batch/time; **`Q`’s row/column convention** (and any `.transpose()` in `_alpha`/`_beta`) must be matched exactly in Rust tests — mirror the reference before claiming parity.

## Completion checklist

- [ ] New module(s) under **`snlds-core`** (e.g. `hmm` / `kernels` — path TBD) with APIs sufficient to run **forward** and **backward** on **`[N,T,K]`** log-evidence tensors + **\(\pi\)** / **`Q`** / **temperature**.
- [ ] Forward pass: **`log_alpha`**, **`log_Z`** consistent with **`_alpha`**.
- [ ] Backward pass: **`log_beta`** consistent with **`_beta`** given **`log_Z`**.
- [ ] **Tests vs reference tensors:** tiny **hand-derived** discrete HMM **or** fixtures (e.g. under `crates/snlds-core/tests/fixtures/` as **`.safetensors`** / **`.npy`**) with asserted **max-abs** or **rtol** tolerance on CPU.
- [ ] **(Optional / phased)** **`_compute_local_evidence`** parity on synthetic latents + Gaussian parameters vs exported reference (can reuse **[M1](M1.md)** `latents_*` for integration-style checks later).
- [ ] **(Optional)** One-shot script to regenerate reference tensors from PyTorch — **not** a CI gate unless promoted.
- [ ] [docs/PRD-burn-port.md](PRD-burn-port.md) **§8.5** updated if new Rust dependencies ship with M2.

## Testing requirements

Inherit **M0** gates: `cargo fmt --check`, `cargo clippy --workspace --all-targets -D warnings`, `cargo test --workspace`.

| Area | Requirement |
|------|--------------|
| **Parity** | **`log_alpha`**, **`log_beta`**, **`log_Z`** match reference within agreed **f32** tolerance (tight on CPU; document chosen `atol`/`rtol` in test code). |
| **Cases** | At least one **small** **`(N,T,K)`** case (e.g. \(N{=}1\), small \(T\), \(K{=}2\) or \(3\)); prefer **no `#[ignore]`** without a tracked issue. |
| **Stability** | Implementations use **log-domain** accumulations (**`logsumexp`**) consistent with PRD [§6](PRD-burn-port.md#6-non-functional-requirements) (numerical stability). |
| **Local evidence** | If implemented in M2: same tolerance policy vs reference for **`[N,T,K]`** log-evidence output. |

**Out of scope:** full **`VariationalSNLDS`**, ELBO, encoder/decoder (**M3+**); Rerun (**[M-Viz](PRD-burn-port.md)**); training CLI (**M4**). Optional Python-in-CI parity script (**not** assumed as a gate unless listed above).
